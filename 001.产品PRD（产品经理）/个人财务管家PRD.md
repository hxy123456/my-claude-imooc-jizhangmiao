# 记账喵 - 产品需求文档（PRD）

## 一、产品概述

### 1.1 产品名称
**记账喵**（CountCat）

### 1.2 产品定位
一款轻量级个人收支记录与管理工具，帮助用户快速记账、清晰掌握每日收支流向，养成良好理财习惯。

### 1.3 目标用户
- **核心用户**：18岁以上个人用户（含家庭、情侣共享记账场景）
- **扩展用户**：小微企业和个体户的简易收支管理
- **用户特征**：追求"简单好用"，不需要复杂财务功能

### 1.4 产品愿景
让记账像发朋友圈一样简单，3秒完成一笔记录。

### 1.5 产品形态规划

| 版本 | 形态 | 说明 |
|------|------|------|
| V1 | Web网页端 | 响应式设计，支持手机浏览器访问 |
| V2 | 移动App + 小程序 | 原生体验，离线支持 |

---

## 二、核心用户场景

| 场景 | 用户行为 | 痛点 |
|------|---------|------|
| 日常消费 | 买完东西想记一笔 | 打开app步骤多，懒得记 |
| 收入记录 | 发工资/收到退款想记一笔 | 只记支出不记收入，结余算不清 |
| 月底复盘 | 想看这个月钱花哪了 | 不知道钱去哪了，心里没数 |
| 分类对比 | 这个月餐饮比上个月多多少 | 没有对比，无法调整消费习惯 |
| 个体户记账 | 进货/营收要分清 | 个人和生意混在一起，算不清 |

## 四、V1 极简MVP 功能详细设计

### 4.1 快速记账

**描述：** 首页即可完成一笔记录，无需跳转页面。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 金额 | 数字 | 是 | 支持小数点后2位，默认弹出数字键盘 |
| 类型 | 枚举 | 是 | 支出 / 收入，默认支出 |
| 分类 | 枚举 | 是 | 两级分类，见4.2 |
| 备注 | 文本 | 否 | 最多50字 |
| 日期 | 日期 | 否 | 默认当天，可回溯补记 |

**交互流程：**
1. 首页点击「+ 记一笔」悬浮按钮
2. 底部弹出记账面板（Bottom Sheet）
3. 金额输入框自动聚焦，数字键盘弹出
4. 选择分类 → 选择子分类（可选）→ 输入备注（可选）
5. 点击「确认记录」→ 轻提示"已记录 ✓"，1.5秒消失

**交互要求：**
- 支出/收入通过顶部Tab切换，默认支出
- 金额输入支持负号自动转收入
- 记账面板支持手势下滑关闭

| 通讯 | 📱 | 话费、会员 |

| 大类 | 图标 | 子分类 |
|------|------|--------|

### 4.5 账单 - 记录列表

**展示规则：**
- 按日分组，每组显示日期 + 当日支出合计 + 当日收入合计
- 每条记录：分类图标 + 分类名 + 备注 + 金额（支出红色，收入绿色）
- 支持按分类筛选（顶部筛选条）
- 支持关键词搜索（搜索备注内容）
- 左滑删除，点击编辑

### 4.6 统计 - 月度数据分析

**展示内容：**

| 模块 | 内容 | 图表形式 |
|------|------|---------|
| 月度总览 | 本月支出 / 本月收入 / 结余 | 大数字 |
| 分类占比 | 各大类支出金额及占比 | 环形图 |
| 日消费趋势 | 每日支出金额 | 柱状图（横轴日期，纵轴金额） |
| 分类排行 | 按支出金额从高到低 | 列表 |

**交互：**
- 左右滑动切换月份
- 点击环形图某分类 → 展示该分类下的记录明细
- 点击柱状图某天 → 展示当天记录明细

## 五、V1.1 增强版功能

### 5.1 预算管理
- 设置月度总预算
- 按分类设置子预算（如：餐饮 ≤ ¥1000）
- 首页显示预算进度条
- 消耗达80%时提醒，100%时强提醒
- 预算进度按分类分别展示

### 5.2 月度环比对比
- 本月 vs 上月各分类支出对比
- 增减金额 + 增减百分比
- 涨跌用颜色区分（涨红跌绿）

### 5.3 月度报告
- 每月1日自动生成上月报告
- 内容：总支出/收入、日均消费、最大单笔、分类Top3、环比变化
- 支持导出为图片分享

## 六、V1.2 效率版功能

### 6.1 周期性自动记账
- 设置每月固定支出（房租/话费/会员等）
- 到期自动生成记录，用户确认后入库
- 支持按月/按周/按日周期

### 6.2 账单导入
- 支持导入微信支付账单CSV
- 支持导入支付宝账单CSV
- 导入时自动匹配分类，用户可批量修正

### 6.3 拍照识别记账
- 拍摄小票/消费截图
- OCR识别金额、商户名
- 用户确认后生成记录

## 九、非功能需求

### 9.1 性能
- 记账操作响应时间 < 200ms
- 列表加载时间 < 500ms
- 统计图表渲染 < 1s

### 9.2 数据存储
- V1：浏览器端 IndexedDB 本地存储
- 数据不收集上传，用户隐私优先
- 单条记录 < 1KB，支持10万条以上记录无性能衰减
- 删除为软删除，支持30天内恢复

### 9.3 兼容性
- 响应式Web，支持手机浏览器和桌面浏览器
- 最低支持：Chrome 80+、Safari 14+、Firefox 80+
- 移动端适配：触控操作、手势滑动

### 9.4 安全
- 数据本地加密存储
- 不收集用户财务数据上传服务器

## 十三、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 记账流程不够快 | 核心体验失败 | 极简3步流程：金额→分类→确认 |
| 分类不匹配用户习惯 | 记账意愿降低 | 两级分类覆盖80%场景，V1.1加自定义 |
| 浏览器数据丢失 | 用户信任崩塌 | 软删除 + 导出备份（V1.1） |
| 功能膨胀 | 失去轻量定位 | 严格按版本迭代，不做投资/借贷等重功能 |

## 十三-A、V1.0.1 用户系统

### 13A.1 注册登录
- 支持用户名 + 密码注册
- 用户名最少2位，密码最少4位
- 密码使用 SHA-256 哈希存储（Web Crypto API）
- 登录态为会话级，关闭浏览器失效（sessionStorage）
- 注册成功后自动登录

### 13A.2 数据隔离
- 每条记账记录绑定 userId
- 所有查询按当前登录用户过滤
- 不同用户看到完全独立的数据
- 新用户首次登录自动填充演示数据

### 13A.3 退出登录
- 首页右上角退出按钮
- 退出后清除登录态和本地缓存
- 跳转到登录页
